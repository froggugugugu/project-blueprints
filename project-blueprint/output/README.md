# Output — AIが生成するアウトプット

このディレクトリにはAIが生成した成果物が出力される。
人間はここの成果物をレビュー・承認する。

## ディレクトリ構成

```text
output/
├── prd/                   PRD（プロダクト要求仕様書）
│   └── PRD_xxx.md
├── design/                アーキテクチャ設計書
│   └── ARCH_xxx.md
├── tasks/                 タスク分解・実装指示書
│   └── TASK_xxx.md
└── reports/               各種レポート
    ├── review/            コードレビューレポート
    ├── test/              テスト結果レポート
    ├── security/          セキュリティスキャンレポート
    └── legal/             法務チェックレポート
```

## フェーズとアウトプットの対応

| フェーズ | 出力先 | 生成スキル | レビュー観点 |
| --- | --- | --- | --- |
| 要件分析 | `prd/` | `/prd` | 要件の網羅性・曖昧さの解消 |
| アーキテクチャ設計 | `design/` | `/architecture` | 技術選定の妥当性・構成の整合性 |
| タスク分解 | `tasks/` | `/plan` | 粒度の適切さ・依存関係の正確性 |
| コードレビュー | `reports/review/` | `/code-review` | 指摘の妥当性 |
| テスト | `reports/test/` | `/e2e-testing` | カバレッジ・テストシナリオの網羅性 |
| セキュリティ | `reports/security/` | `/security-scan` | 脆弱性の重要度・対応方針 |
| 法務チェック | `reports/legal/` | `/legal-check` | リスクレベル・対応要否 |

## レビュー後のアクション

- **承認**: PJMに承認を伝え、次のフェーズに進む
- **差し戻し**: 具体的なフィードバックを添えて修正を依頼する
- **保留**: 追加情報が必要な場合、`input/`に補足メモを追加する
