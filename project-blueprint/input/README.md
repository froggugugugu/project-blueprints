# Input — 人間が作成するインプット

このディレクトリには**人間が作成・編集する**ファイルを配置する。
AIはここのファイルを読み取り専用で参照し、`output/`に成果物を生成する。

## ディレクトリ構成

```text
input/
└── requirements/          要求メモ・要件メモ
    ├── REQ_001_xxx.md     機能ごとの要求メモ
    ├── REQ_002_xxx.md
    └── ...
```

## 要求メモの書き方

自由形式で構わないが、以下の情報があるとAIの出力品質が上がる:

- **何を作りたいか**（機能の概要）
- **誰が使うか**（ユーザー像）
- **なぜ必要か**（背景・目的）
- **制約・条件**（技術的制約、期限、優先度）

箇条書き、走り書き、会議メモの転記でも可。AIが構造化する。

## ファイル命名規則

```text
REQ_<連番>_<機能名>.md
```

例: `REQ_001_user-auth.md`, `REQ_002_dashboard.md`
