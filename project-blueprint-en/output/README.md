# Output — AI-Generated Deliverables

This directory contains deliverables generated by AI.
Humans review and approve the deliverables here.

## Directory Structure

```text
output/
├── prd/                   PRD (Product Requirements Document)
│   └── PRD_xxx.md
├── design/                Architecture Design Documents
│   └── ARCH_xxx.md
├── tasks/                 Task Decomposition / Implementation Specifications
│   └── TASK_xxx.md
└── reports/               Various Reports
    ├── review/            Code Review Reports
    ├── test/              Test Result Reports
    ├── security/          Security Scan Reports
    └── legal/             Legal Check Reports
```

## Phase and Output Mapping

| Phase | Output Location | Generating Skill | Review Focus |
| --- | --- | --- | --- |
| Requirements Analysis | `prd/` | `/prd` | Completeness of requirements, resolution of ambiguities |
| Architecture Design | `design/` | `/architecture` | Validity of technology choices, structural consistency |
| Task Decomposition | `tasks/` | `/plan` | Appropriate granularity, accuracy of dependencies |
| Code Review | `reports/review/` | `/code-review` | Validity of findings |
| Testing | `reports/test/` | `/e2e-testing` | Coverage, completeness of test scenarios |
| Security | `reports/security/` | `/security-scan` | Severity of vulnerabilities, response strategy |
| Legal Check | `reports/legal/` | `/legal-check` | Risk level, need for action |

## Post-Review Actions

- **Approve**: Inform the PJM of the approval and proceed to the next phase
- **Request Changes**: Provide specific feedback and request revisions
- **Hold**: If additional information is needed, add supplementary notes to `input/`
