# TEAMCREATE 

> **用途**: 機能ごとにこのファイルをコピーし、`TEAMCREATE_<機能名>.md` として保存する  
> **チーム構成**: PL（プロジェクトリーダ）・開発者・レビュア・テスタの4役割  
> **対象**: React / Next.js フロントエンドアプリ
> **このファイルを利用した指示サンプル指示**:.claude/teams/TEAMCREATE_ASSIGNMENT.md を読み、ファイル内のすべての指示に従ってください。 指定どおりにエージェントチームを作成し、委任モードを使用し、 変更を行う前に計画の承認を求めてください。
> **Read .claude/teams/TEAMCREATE_ASSIGNMENT_EN.md and follow all instructions in that file. Create the agent team as specified, use delegate mode, and require plan approval before any changes.**

---
## 1. チーム構成

| 役割 | 担当内容 | モデル |
|---|---|---|
| **PL（プロジェクトリーダ）** | 機能全体の整合性確認、他機能との連携調整、計画承認実装はしない（delegate） | Opus（Lead） |
| **開発者** | 機能実装 + ユニットテスト作成 | Sonnet |
| **レビュア** | コード品質・設計方針・規約準拠のレビュー | Sonnet |
| **テスタ** | 結合テスト + Playwright E2E テスト作成・実行 | Sonnet |

## 2. タスク依存関係

```
開発者: TASK md を読んで実装 + ユニットテスト
    ↓（実装完了）
レビュア: コードレビュー実施、指摘事項を開発者にフィードバック
    ↓（レビュー承認）
テスタ: 結合テスト + Playwright E2E テスト作成・実行
    ↓（テスト完了）
PL: 全体確認、他機能との整合性チェック、完了判定
```

## 3. Claudeへの指示

以下を Claude Code に貼り付けて実行してください。

**📎 Claudeへの指示**

**📎 和訳**

```
[assignment] 機能のエージェントチームを作成してください。
変更前に必ず計画承認を要求すること。
デリゲートモードで動作すること — あなたはPLです。自分でコードを実装しないでください。

3人のチームメイトを生成してください：

- 開発者:
  .claude/tasks/TASK03_EN.md を読み、その仕様に従って実装すること。
  実装と同時にユニットテストも作成すること。
  作業範囲は/のみ。
  ユニットテストには React Testing Library + vitest を使用すること。
  変更前に必ず計画承認を要求すること。

- レビュア:
  開発者の実装完了を待つこと。
  開発者の全変更を以下の観点でレビューすること：
    - コード品質と可読性
    - CLAUDE.md のコーディング規約への準拠
    - コンポーネント設計（適切な分離、再利用性）
    - アクセシビリティ（a11y）への配慮
    - パフォーマンス懸念（不要な再レンダリング、メモ化の欠落）
  具体的で対応可能なフィードバックを開発者に送ること。
  ソースコードは一切変更しないこと。

- テスタ:
  レビュアが開発者のコードを承認するまで待つこと。
  .claude/tasks/TASK03_EN.md を読み、期待される動作を理解すること。
  [e2e/assignment] に結合テストと Playwright E2E テストを作成すること。
  カバー範囲: 正常系、エラーケース、エッジケース、レスポンシブ動作。
  全テストを実行し、結果をPLに報告すること。
  ソースコードは変更しないこと — テストファイルの作成・変更のみ。

タスク依存関係：
1. 開発者の実装完了 → レビュアがレビュー開始
2. レビュア承認 → テスタがテスト開始
3. テスタ完了 → PLが最終確認

PLの責務（あなた）：
- 各チームメイトの計画を確認し、承認または却下する
- 他機能やアプリ全体のアーキテクチャとの整合性を確保する
- ユニットテストを含まない開発者の計画は承認しないこと
- 全テストがパスするまで機能完了としないこと
```
